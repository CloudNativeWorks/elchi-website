# Elchi Documentation

Elchi is an enterprise **Envoy management platform**: a central control plane with a
visual editor for every Envoy resource, plus edge agents that deploy and operate
Envoy on your hosts — with GSLB, ACME certificates, WAF, API discovery, and API
security built in.

## Start here

- **[Introduction](/getting-started/introduction)** — what Elchi is and how the pieces fit together.
- **[Architecture](/getting-started/architecture)** — the processes, the edge node, and how every wire connects.
- **[Quick Start](/getting-started/quickstart)** — get the platform running on Kubernetes in four steps.

## Install it

Pick the deployment substrate that fits — all bring up the same control plane:

- **[Helm](/installation/helm-platform/overview)** — on an existing Kubernetes cluster.
- **[Docker Swarm](/installation/docker-swarm/overview)** — one command, container-based, single-node or HA, online or air-gapped.
- **[Single-Host (kind)](/installation/kind-quickstart)** — the whole stack on one VM for evaluation.
- **[Bare-Metal](/installation/bare-metal/overview)** — plain VMs with systemd, no containers.
- **[Elchi Client](/installation/client/overview)** — then install the edge agent (+ [bundled Shield](/installation/client/shield-sidecar)) next to Envoy.
